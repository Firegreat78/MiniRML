Инструментальная система разработки программ управления промышленными роботами. Разработана с использованием Qt 6.8.2
# Краткое описание проекта
В данном проекте:
- разработан интерпретируемый C-подобный язык программирования для управления простым роботом-манипулятором;
- разработан графический интерфейс, показывающий отладочную информацию (текущие значения переменных в данной области видимости и типы этих переменных; координаты шарниров в пространстве; выполненные шаги выполнения).

Робот-манипулятор, управляемый языком программирования, представляет собой совокупность шарниров, соединённых прямыми сегментами. Исходное состояния робота-манипулятора с 6 сегментами представлено на изображении ниже.

## Типы данных
Язык программирования имеет следующие типы данных:
- `short` - знаковое целое число, занимающее 2 байта. Число, представляемое этим типом данных, может принимать значение в диапазоне $[-2^{15}; 2^{15}-1]$ = $[-32,768; 32,767]$;

- `int` - знаковое целое число, занимающее 4 байта. Число, представляемое этим типом данных, может принимать значение в диапазоне $[-2^{31}; 2^{31}-1]$ = $[-2,147,483,648; 2,147,483,647]$;

- `long` - знаковое целое число, занимающее 8 байт. Число, представляемое этим типом данных, может принимать значение в диапазоне $[-2^{63}; 2^{63}-1]$ = $[-9,223,372,036,854,775,808; 9,223,372,036,854,775,807]$;

- `double` - число с плавающей запятой двойной точности по IEEE 754;

- `bool` - булевый тип.

### Особенности типов литералов
Целочисленные литералы (и десятичные, и шестнадцатиричные):
- со значением, принадлежащем диапазону $[-2^{15}; 2^{15}-1]$, имеют тип `short`;

- со значением, не принадлежащем диапазону $[-2^{15}; 2^{15}-1]$, но принадлежащем диапазону $[-2^{31}; 2^{31}-1]$, имеют тип `int`;

- со значением, не принадлежащем диапазону $[-2^{31}; 2^{31}-1]$, но принадлежащем диапазону $[-2^{63}; 2^{63}-1]$, имеют тип `long`;

Литералы `true` и `false` имеют тип `bool`.

Литералы с точкой (такие, как `0.0`, `.0`, `0.`, и другие) или с символом `e` (такие, как `0e0`, )

### Особенности приведения типов
Реализовано приведение типов между целочисленными типами данных (`short`, `int`, `long`).

Control flow реализован с использованием операторов `if` и `while`.

## Встроенные функции
В языке имеются следующие встроенные функции:
- функция `pi()` возвращает число `π` в максимально возможной точности для формата double-precision IEEE 754: `0x400921fb54442d18`.

- функция `sin(x)` возвращает синус угла. Аргумент `x` имеет тип `double` и описывает угол в радианах.

- функция `cos(x)` возвращает косинус угла. Аргумент `x` имеет тип `double` и описывает угол в радианах.

- функция `tan(x)` возвращает тангенс угла. Аргумент `x` имеет тип `double` и описывает угол в радианах. Если `cos(x)` равен нулю, произойдёт ошибка интерпретации.

- функция `ctg(x)` возвращает котангенс угла. Аргумент `x` имеет тип `double` и описывает угол в радианах. Если `sin(x)` равен нулю, произойдёт ошибка интерпретации.

- функция `arcsin(x)` возвращает арксинус угла. Аргумент `x` имеет тип `double`. Возвращаемое значение представляет угол в радианах в диапазоне $[-\frac{\pi}{2}; \frac{\pi}{2}]$. Если число `x` не принадлежит отрезку $[-1; 1]$, произойдёт ошибка интерпретации.

- функция `arccos(x)` возвращает арккосинус угла. Аргумент `x` имеет тип `double`. Возвращаемое значение представляет угол в радианах в диапазоне $[0; \pi]$. Если число `x` не принадлежит отрезку $[-1; 1]$, произойдёт ошибка интерпретации.

- функция `arctan(x)` возвращает арктангенс угла. Аргумент `x` имеет тип `double`. Возвращаемое значение представляет угол в радианах в интервале $(-\frac{\pi}{2}; \frac{\pi}{2})$.

- функция `arcctg(x)` возвращает арккотангенс угла. Аргумент `x` имеет тип `double`. Возвращаемое значение представляет угол в радианах в интервале $(0; \pi)$.

- функция `atan2(y, x)` возвращает арктангенс отношения `y/x` с учётом знаков обоих аргументов. Аргументы `y` и `x` имеют тип `double`. Возвращаемое значение представляет угол в радианах между положительной осью X и точкой `(x, y)` в диапазоне $[-\pi; \pi]$. Если оба аргумента равны нулю, произойдёт ошибка интерпретации.

- функция `deg2rad(x)` преобразует градусы в радианы. Аргумент `x` имеет тип `double` и представляет угол в градусах. Возвращает угол в радианах.

- функция `rad2deg(x)` преобразует радианы в градусы. Аргумент `x` имеет тип `double` и представляет угол в радианах. Возвращает угол в градусах.

- функция `compare_double(lhs, rhs, decimal)` сравнивает два числа типа `double` с заданной точностью. Аргументы `lhs` и `rhs` имеют тип `double`, аргумент `decimal` имеет целочисленный тип (`short`/`int`/`long`). Возвращает значение типа `short`: `-1`, если `lhs < rhs`; `0`, если `lhs ≈ rhs` с точностью до `decimal` знаков после запятой; `1`, если `lhs > rhs`. Количество десятичных знаков не может быть отрицательным.

## Встроенные команды для управления позиционированием робота-манипулятора
Для управления роботом-манипулятором используются команды `Move` и `Rotate`.

Команда `Move(x, y)` поступательно сдвигает все шарниры робота. Аргументы, передаваемые в команду, должны иметь тип `double`.

Описание аргументов:
- `x` - смещение по оси X;
- `y` - смещение по оси Y.

Команда `Rotate(i, nX, nY, nZ, φ)` поворачивает робота за шарнир на угол `|φ|` в радианах вокруг некоторой оси согласно правилу: 
- правой руки, если φ > 0;
- левой руки, если φ < 0;

Описание аргументов:
- `i` - индекс шарнира, за который осуществляется поворот (тип аргумента: `short`);
- `nX` - компонента X направляющего вектора оси, вокруг которой осуществляется поворот других шарниров (тип аргумента: `double`);
- `nY` - компонента Y направляющего вектора оси, вокруг которой осуществляется поворот других шарниров (тип аргумента: `double`);
- `nZ` - компонента Z направляющего вектора оси, вокруг которой осуществляется поворот других шарниров (тип аргумента: `double`);
- `φ` - угол поворота в радианах (тип аргумента: `double`).

### Графическая иллюстрация поворота

Приведём графическую иллюстрация работы функции для поворота рук робота. Для простоты иллюстрации предположим, что робот расположен в двумерном пространстве (следовательно, вращение возможно только строго вокруг оси Z). В качестве направления оси возьмём положительное направление, следовательно, nX = `0.0`, nY = `0.0`, nZ может иметь любое положительное значение, для примера возьмём значение `1`.

Заметим, что при повороте координаты первого шарнира остались неизменны, а координаты последующих шарниров поменялись (а именно, они все повернулись вокруг оси, проходящий через шарнир с индексом 0, на угол `π / 4.0`, согласно правилу правой руки).
Далее, вслед за первым поворотом, можем повернуть робота за второй шарнир на 45 градусов по часовой стрелке. Получим следующий результат:

Заметим, что при повороте координаты первого и второго шарниров остались неизменны, а координаты последующих шарниров снова поменялись (а именно, они все повернулись вокруг оси, проходящий через шарнир с индексом `1`, на угол `π / 4.0`, согласно правилу левой руки).
Наконец, повернём робота за третий шарнир на 90 градусов по часовой стрелке, и получим следующий результат:

Для совершения данной операции необходимо будет в коде вызвать `Rotate(2, 0.0, 0.0, 1.0, -pi() / 2.0)`.

Заметим, что при повороте координаты первого, второго, и третьего шарниров остались неизменны, а координаты последнего шарнира поменялись.

Код, написанный на интерпретируемом языке, который был использован для демонстрации поворота:

```
void main()
{
    Rotate(0, 0.0, 0.0, 1.0, pi() / 4.0);
    Rotate(1, 0.0, 0.0, 1.0, -pi() / 4.0);
    Rotate(2, 0.0, 0.0, 1.0, -pi() / 2.0);
}
```

Скриншот из, непосредственно, инструментальной системы:

<img width="1810" height="991" alt="image" src="https://github.com/user-attachments/assets/84f3df13-8a06-49b1-8068-4dd7b9c15df6" />

Итак, во всех случаях имеем поворот точек вокруг оси, проходящей через шарнир, за который осуществляется поворот.

Для подсчёта новых координат используется формула поворота Родрига в трёхмерном пространстве.

Пусть точка до поворота имеет координаты (`x`, `y`, `z`), а после поворота имеет координаты (`x'`, `y'`, `z'`). Вращение происходит вокруг оси с направляющим вектором (`u_x`, `u_y`, `u_z`), проходящей через точку (`p_x`, `p_y`, `p_z`). 

Тогда вычисление новых координат `(x', y', z')` производится по следующим уравнениям:

```math
\begin{aligned}
x' &= p_x + x \cdot \cos(\theta) + (u_y Z - u_z Y) \cdot \sin(\theta) + u_x (u_x X + u_y Y + u_z Z)(1 - \cos(\theta)) \\
y' &= p_y + y \cdot \cos(\theta) + (u_z X - u_x Z) \cdot \sin(\theta) + u_y (u_x X + u_y Y + u_z Z)(1 - \cos(\theta)) \\
z' &= p_z + z \cdot \cos(\theta) + (u_x Y - u_y X) \cdot \sin(\theta) + u_z (u_x X + u_y Y + u_z Z)(1 - \cos(\theta))
\end{aligned}

```

Где

```math
$$
X = x - p_x, \quad Y = y - p_y, \quad Z = z - p_z
$$
```
